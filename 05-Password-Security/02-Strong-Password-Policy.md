# Strong Password Policy
## Core Concept
A strong password policy is a set of rules to create and maintain secure passwords, preventing unauthorized access through brute-force or guessing attacks.

## Why It Matters
Weak passwords are the #1 cause of account breaches. A strong policy is the first line of defense for all user accounts.

## How It Works (Theory)
A strong password policy enforces:
1. **Length**: Minimum 12 characters (longer = more secure).
2. **Complexity**: Mix of uppercase, lowercase, numbers, and special characters.
3. **Uniqueness**: No reuse of passwords across multiple accounts.
4. **Expiration**: Regular password rotation (every 90 days for critical accounts).
5. **Prohibition**: No common passwords, dictionary words, or personal information.

## Practical Example
- **Weak password**: `Password123` (common, short, easy to guess).
- **Strong password**: `K9$pX7!zR2@qL` (long, complex, unique, no personal info).
- **Best practice**: Use a password manager to generate and store unique, strong passwords.

## How to Defend / Secure
1. Enforce minimum 12-character passwords for all accounts.
2. Block common passwords and password reuse.
3. Mandate multi-factor authentication (MFA) for all accounts.
4. Use a password manager to eliminate weak, reused passwords.
5. Regularly audit user passwords for compliance.

## Key Takeaways
1. Length is more important than complexity for password security.
2. Password managers are the best way to enforce strong, unique passwords.
3. MFA adds a critical second layer of defense even if passwords are compromised.
